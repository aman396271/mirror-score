# DELIVERY STANDARD — 工程交付标准

> 本文档定义项目的阶段划分、节点标准、完成度分级、角色职责、返工条件。
> 所有任务的派发、执行、验收必须基于本标准。

---

## 一、项目边界

### 当前阶段允许做
- Web 站点前后端骨架
- 前端页面骨架（首页、上传页、结果页）
- 后端 API 骨架（健康检查、mock analyze 接口）
- 图片上传交互
- mock 分析结果返回与展示
- mock 付费解锁 UI
- 文档与工程规范
- 本地运行验证

### 当前阶段禁止做
- 真实模型训练或接入
- 人脸识别身份确认
- 真实支付接入
- 用户登录/注册系统
- 用户历史记录
- 云部署/生产级监控
- 推荐系统
- 社会价值评分、受欢迎程度等结论
- 未成年人分析
- 多图比对
- App 端开发

### 允许的技术简化
- 图像分析用 mock 数据
- 支付用 mock 状态
- 数据库后补
- 不做对象存储
- 不做鉴权
- 不做复杂样式系统

### 新增需求准入条件
任何新需求必须同时满足以下全部条件才可进入任务板：
1. 不破坏当前阶段目标
2. 不引入新技术栈
3. 不增加高复杂依赖
4. 不影响现有验收标准
5. 能在当前节点之后自然接入

## 二、阶段与节点定义

### 节点 A：项目地基完成
**内容：** 文档体系、前后端目录、README、.gitignore、启动方式约定、Claude→Codex 工作流固定
**完成标准：**
- 文档齐全（PROJECT_OVERVIEW / TASK_BOARD / ACCEPTANCE_CRITERIA / CHANGELOG / DELIVERY_STANDARD）
- 任务板可维护
- Codex 能被 Claude 正确同步派单并返回结果
- 最小文件结构落地

### 节点 B：前端骨架可运行
**内容：** frontend/ 为 Next.js 14+ App Router + TypeScript + Tailwind，首页可访问
**完成标准：**
- npm install 成功
- npm run dev 成功，localhost:3000 可访问
- 页面显示项目名称和描述，无报错
- npm run build 和 npm run lint 通过

### 节点 C：后端骨架可运行
**内容：** backend/ 含 FastAPI app，requirements.txt，main.py
**完成标准：**
- pip install -r requirements.txt 成功
- uvicorn main:app --reload 可启动，端口 8000
- GET / 返回 {"status":"ok","service":"MirrorScore API"}
- GET /health 返回 {"status":"healthy"}

### 节点 D：Phase 0 总体验收（⚠️ 大节点 — 停下来向用户汇报）
**验收内容：** 目录结构、前端启动证据、后端启动证据、/health 响应、文件列表、文档更新
**用户只需确认：** 前后端能跑、文档够用、结构清晰

### 节点 E：上传页完成
**完成标准：**
- 支持 jpg/png/webp，文件大小限制可用
- 上传前后状态清晰，预览图可用
- 错误提示清晰
- 移动端可用

### 节点 F：后端 /analyze mock 接口完成
**完成标准：**
- POST /analyze 可接收请求
- 返回结构包含：scoreTotal, ageRange, subScores, freeSummary, isLocked
- 输入验证和错误处理完整

### 节点 G：结果页完成
**完成标准：**
- 总分卡片、年龄区间、五维分项、免费建议、锁定内容提示、解锁按钮
- mock 数据可稳定渲染，无致命报错

### 节点 H：主流程跑通（⚠️ 大节点 — 停下来向用户汇报）
**验收内容：** 上传→分析→结果全流程可演示
**用户只需确认：** 流程连贯、mock 数据完整

### 节点 I：付费解锁 UI 完成（⚠️ 大节点 — 停下来向用户汇报）
**完成标准：**
- 不接真实支付，但视觉和交互上具备商业化演示能力
- 免费/锁定/解锁层级清楚

## 三、完成度分级

每个节点必须按以下 4 级评判，只有 ACCEPTED 才能标记 DONE：

| 等级 | 定义 |
|------|------|
| NOT STARTED | 未开始，无代码无结构 |
| PARTIAL | 有部分文件或代码，但不能运行或不能验收 |
| FUNCTIONAL | 基本能运行，核心目标达成，可继续推进 |
| ACCEPTED | Claude 验收通过，证据齐全，文档已更新 |

---

## 四、角色职责

### Claude 必须做到
- 只管理、拆任务、验收、返工
- 不自己写大量业务代码
- 每次只派一个小任务，带明确边界
- 任务完成后必须让 Codex 自测
- 自测失败必须返工，不打扰用户
- 不到大节点不打扰用户
- 大节点时做完整汇报
- 遇到难解决的问题，联网搜索解决方案后自行解决

### Claude 禁止做
- 自己大段实现产品代码
- 擅自新增需求
- 一次性派很多任务
- 任务没验证就宣布完成
- 因为"看起来差不多"就放行

### Codex 必须做到
- 只执行当前任务，不扩展范围
- 修改后立刻运行验证命令
- 输出：修改文件列表、启动方式、测试结果、已知限制

### Codex 禁止做
- 改无关文件
- 扩大需求
- 未验证就声称完成

---

## 五、返工条件

以下任一情况必须返工：
- 跑不起来
- 文档与实际不一致
- 接口字段不一致
- 目录结构混乱
- 任务范围被擅自扩大
- 缺少验证命令输出
- 缺少回传证据
- UI 能看但流程断裂
- 依赖缺失
- 修改了无关文件

同一任务最多返工 3 次，3 次后升级给用户决策。
